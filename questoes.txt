1. O que é um repositório no Git e qual a sua importância?
    É o lugar onde você pode armazenar o seu código e arquivos, sendo importante por permitir que os desenvolvedores acessem as verões do código, podendo colaborar e acompanhar as alterações

2. Qual a função do Git Bash?
    O Git Bash permite usar os comandos do Git no sistema Windows, emulando um terminal Unix

3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
    O git add <arquivo> é para realizar a selação individual dos arquivos, já o git add . seleciona todos os itens para serem commitados.

4. Por que utilizamos o ponto (.) no comando git add .?
    O ponto (.) é indica ao comando que ele deverá realizar a seleção de todos os itens para serem commitados

5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.
    O git commit é como se fosse uma gravação do momento, onde será registrado essa gravação com uma mensagem para indicar o que é essa gravação, além disso é importante escrever mensagens descritivas para ser mais facil identificar as alterações realizadas no seu projeto.

6. Qual a função do comando git push?
    O git push é um comando de confirmação para os commits serem enviados ao repositorio registrado no GitHub

7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
    Para ser mais facil identificar erros e criar uma branch organizada, com mensagens se referindo a cada parte do projeto

8. Explique a função das tags HTML e como elas estruturam uma página.
    Todas as funções do HTML envolvem principalmente a estrutura e o conteudo da página que está sendo criada

9. O que são atributos em uma tag HTML e qual a sua função?
    São informações que controlam o comportamento da tag, para que a tag se comporte da maneira esperada

10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?
    Emmet é um conjunto de plug-ins que auxiliam na digitação de códigos, agilizando eles por meio de um assistente de conteúdo

11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?
    O box-sizing é um comando que manipula a propriedade padrão da box model, podendo emular o comportamento dos navegadores que não suportam a especificação da box model do CSS, usamos ele no reset inicial para facilitar a manipulação dos elementos da página, não precisando calcular o tamanho da div por conta de um padding que foi colocado antes.

12. Por que é importante utilizar tags semânticas no HTML?
    Para melhorar a estrutura do site e organiza-lo melhor

13. O que são media queries no CSS e qual a sua função?
    Media queries é o recurso que serve para adaptar o site a diferentes dispositivos, facilitando a visualização em uma diversidade de dispositivos

14. Qual a função do GitHub Pages?
    É um jeito que podemos publicar o site, para visualização, nos servidores do próprio GitHub.

15. O que é o Flexbox no CSS e para que ele é utilizado?
    O Flexbox é um dos jeitos de organizar os elementos e montar layouts flexiveis, ele é utilizando principalmente quando quer organizar os elementos que estão em um container, alinhando eles ao meio ou então distanciando os elementos.

16. Qual a diferença entre justify-content e align-items em um container Flexbox?
    No container Flexbox, o justify-content serve para alinhar os itens ao longo da flex-direction, podendo adicionar espaço entre os itens e alinhar os itens no começo, no meio ou no fim. O align-items alinha os itens independente do tamanho deles, podendo estica-los e centraliza-los no meio do container.

17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
    O flex-grow serve para definir a quantidade de espaço que os itens poderão ocupar no container, dependendo do numero o item irá ocupar um espaço maior do que os outros no container, ou seja, o flex grow é um meio para utilizar bem o espaço do container, além de só funcionar em uma Flexbox.

18. Explique a diferença entre flex-direction: column; e flex-direction: row;
    Flex-direction: column define a direção vertical para os itens, ou seja, uma coluna, já o flex-direction: row define a direção horizontal para os itens, ou seja, uma linha.
